# `@pruvious/i18n`

`@pruvious/i18n` is a small, dependency-free internationalization library. It powers Pruvious's translation system and works on its own anywhere you need translations with placeholders and conditional plurals.

## Installation

```sh
npm install @pruvious/i18n
```

## Basic usage

```ts
import { I18n } from '@pruvious/i18n'

const i18n = new I18n()

i18n.defineTranslatableStrings({
  domain: 'default',
  language: 'de',
  strings: {
    'Welcome': 'Willkommen',
    'Hello, $name!': 'Hallo, $name!',
  },
})

i18n._('de', 'Welcome')                    // 'Willkommen'
i18n._('de', 'Hello, $name!', { name: 'Alice' }) // 'Hallo, Alice!'
```

The `_(language, key, placeholders?)` method looks up a string in the `default` domain. Placeholder names in the key match keys in the second argument.

## Domains

Strings can be grouped into domains - for example, one per package or per feature area:

```ts
i18n.defineTranslatableStrings({
  domain: 'errors',
  language: 'de',
  strings: {
    'Not found': 'Nicht gefunden',
  },
})

i18n.__('errors', 'de', 'Not found') // 'Nicht gefunden'
```

`__(domain, language, key, placeholders?)` is the two-argument form. `_` is the same call against the `'default'` domain.

## Patterns and conditions

For plurals and conditional output, use `createPattern`:

```ts
import { createPattern } from '@pruvious/i18n'

i18n.defineTranslatableStrings({
  domain: 'default',
  language: 'de',
  strings: {
    'Displayed: $count entries': createPattern(
      'Angezeigt: $count $entries',
      { count: 'number' },
      {
        entries: [
          { conditions: [{ count: 1 }], output: 'Eintrag' },
          'Einträge',
        ],
      },
    ),
  },
})

i18n._('de', 'Displayed: $count entries', { count: 1 }) // 'Angezeigt: 1 Eintrag'
i18n._('de', 'Displayed: $count entries', { count: 5 }) // 'Angezeigt: 5 Einträge'
```

The signature is `createPattern(translation, input, replacements?)`.

- **`translation`** is the output string with `$placeholder` markers.
- **`input`** declares the type of each top-level placeholder (`'number'`, `'string'`, ...).
- **`replacements`** maps placeholder names to a list of conditional outputs. Each entry is either a static string or `{ conditions, output }`. The first matching branch wins.

Conditions support equality (`{ count: 1 }`) and operators (`{ count: { '>': 99 } }`).

## Complex conditions

```ts
i18n.defineTranslatableStrings({
  domain: 'default',
  language: 'en',
  strings: {
    '$count items': createPattern(
      '$count $items',
      { count: 'number' },
      {
        count: [
          { conditions: [{ count: 0 }], output: 'No' },
          { conditions: [{ count: { '>': 99 } }], output: '99+' },
          '$count',
        ],
        items: [
          { conditions: [{ count: 1 }], output: 'item' },
          'items',
        ],
      },
    ),
  },
})

i18n._('en', '$count items', { count: 0 })   // 'No items'
i18n._('en', '$count items', { count: 1 })   // '1 item'
i18n._('en', '$count items', { count: 99 })  // '99 items'
i18n._('en', '$count items', { count: 500 }) // '99+ items'
```

Notice that both the count display and the noun pluralization use the same `count` variable - each placeholder picks the first branch whose conditions match.

## The `I18n` class

| Method | Description |
| :--- | :--- |
| `defineTranslatableStrings({ domain, language, strings })` | Register a batch of strings for a domain + language. |
| `_(language, key, placeholders?)` | Translate a key in the `default` domain. |
| `__(domain, language, key, placeholders?)` | Translate a key in a specific domain. |
| `_$(language, key, placeholders?)` | Loosely-typed variant of `_` (skips static handle/placeholder checks). |
| `__$(domain, language, key, placeholders?)` | Loosely-typed variant of `__`. |
| `hasDefinition(domain, language)` | `true` if any string is registered for the domain + language. |
| `getDefinition(domain, language)` | Returns the registered strings, or `undefined`. |
| `setFallbackLanguage(language)` | Replace the fallback chain with a single language. |
| `setFallbackLanguages(languages)` | Replace the fallback chain with the given ordered list. |
| `clone(deep?)` | Returns a copy of the I18n instance. Set `deep: true` to also clone internal data structures. |

## How Pruvious uses it

Pruvious creates a singleton `i18n` instance on the server (`#pruvious/server`) and a separate one on the client (`#pruvious/app/i18n`). Translatable strings collected from `server/translations/` are merged in at build time. From any Vue component you can call `__('domain', 'key', placeholders?)` to translate.

See [Translation composables (`__`, `_`, `useLanguage`)](../api/composables.md#translations) for the Vue integration.
