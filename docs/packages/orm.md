# `@pruvious/orm`

`@pruvious/orm` is the typed ORM that Pruvious uses under the hood. It is a standalone package - you can use it without the rest of Pruvious whenever you need a small, opinionated ORM with first-class support for SQLite, PostgreSQL, and Cloudflare D1.

If you are building a Pruvious site, you usually interact with the ORM through collections and `selectFrom('Articles')`. This page covers using the package directly.

## Installation

```sh
npm install @pruvious/orm
```

For PostgreSQL, also install `pg`:

```sh
npm install pg
```

## Quick start

```ts
import { Collection, Database, Field, textFieldModel } from '@pruvious/orm'

const db = new Database({
  driver: 'sqlite://database.sqlite',
  collections: {
    Users: new Collection({
      fields: {
        email: new Field({ model: textFieldModel(), options: {}, required: true, nullable: false }),
        role:  new Field({ model: textFieldModel(), options: {} }),
      },
      indexes: [{ fields: ['email'], unique: true }],
      foreignKeys: [
        {
          field: 'role',
          referencedCollection: 'Roles',
          referencedField: 'name',
          action: ['ON UPDATE RESTRICT', 'ON DELETE SET NULL'],
        },
      ],
    }),
    Roles: new Collection({
      fields: {
        name: new Field({ model: textFieldModel(), options: {} }),
      },
      indexes: [{ fields: ['name'], unique: true }],
    }),
  },
  sync: { dropNonCollectionTables: true, dropNonFieldColumns: true },
})

await db.connect() // Connects and syncs the schema.
```

## Drivers

The `driver` option accepts a connection string or a driver instance.

### SQLite

Powered by [`better-sqlite3`](https://www.npmjs.com/package/better-sqlite3). Path is relative to the working directory:

```ts
new Database({ driver: 'sqlite://relative/path/to/database.sqlite' })
```

Pass an instance directly when you need custom pragma:

```ts
import Database from 'better-sqlite3'

new Database({ driver: new Database('../tmp/pruvious.sqlite') })
```

### PostgreSQL

```ts
new Database({
  driver: 'postgres://user:pass@localhost:5432/pruvious?ssl=true',
  PGPool: pg.Pool, // required when using a connection string
})
```

Or pass an instance:

```ts
import pg from 'pg'

new Database({
  driver: new pg.Pool({ connectionString: 'postgres://admin:12345678@localhost:5432/pruvious' }),
})
```

### Cloudflare D1

Inside a Worker:

```ts
export default {
  async fetch(request, env, ctx) {
    const db = new Database({ driver: env.DB })
    // ...
  },
}
```

Or by binding name string:

```ts
new Database({ driver: 'd1://DB' })
```

D1 does **not** support transactions - `db.transaction()` will run statements in order without the ability to roll back. Plan for partial failures.

## Collections

A `Collection` abstracts a table. It is defined by its `fields` map and optional metadata:

```ts
const Users = new Collection({
  fields: {
    email: new Field({ model: textFieldModel(), required: true, nullable: false, options: {} }),
    name:  new Field({ model: textFieldModel(), options: {} }),
  },
})

typeof Users.TColumnNames        // 'email' | 'name'
typeof Users.TRequiredColumns    // 'email'
typeof Users.TNullableColumns    // 'name'
typeof Users.TCastedTypes.email  // string
```

Optional metadata: `key` (stable id for sync), `indexes`, `foreignKeys`, `hooks`.

### The `Options` table

Every `Database` has an automatically created `Options` table for internal sync state and key-value storage:

```ts
await db.setOption('mySettings', { foo: 'bar' })
await db.setOptions({ mySettings: { foo: 'BAR' }, myTitle: 'BAZ' })

await db.getOption('mySettings')      // { foo: 'BAR' }
await db.getOption<string>('myTitle') // 'BAZ'
await db.getAllOptions()              // { mySettings: { foo: 'BAR' }, myTitle: 'BAZ' }

await db.deleteOption('mySettings')   // true
await db.deleteAllOptions()           // number of options removed
```

There is also a simple lock primitive used by sync:

```ts
await db.lock('foo')      // true on success
await db.isLocked('foo')  // true
await db.unlock('foo')
```

## Fields

A `Field` represents a column. Each field needs a [field model](#field-models) and `options`:

```ts
const address = new Field({
  model: textFieldModel(),
  required: true,
  nullable: false,
  options: {},
})
```

Optional properties:

- `key` - stable id used by sync.
- `nullable` - default `true`.
- `required` - default `false`.
- `immutable` - default `false`.
- `autoGenerated` - default `false`.
- `default` - default value override.
- `conditionalLogic` - control whether `required` applies based on other fields.
- `validators` - additional validation callbacks.
- `inputFilters` - rewrite the input before insert/update.

### Field models

Built-in models, each handling serialization, validation, and TypeScript typing:

| Model | Stored as | Decoded as |
| :--- | :--- | :--- |
| `arrayFieldModel` | `text` | `Array` of primitives |
| `bigIntFieldModel` | `bigint` | `number` |
| `booleanFieldModel` | `boolean` | `boolean` |
| `junctionFieldModel` | junction table | `number[]` (record ids) |
| `matrixFieldModel` | `text` | `Array` of primitives (numbers or strings) |
| `numberFieldModel` | `numeric` | `number` |
| `objectFieldModel` | `text` | `object` |
| `repeaterFieldModel` | `text` | `Array<object>` |
| `structureFieldModel` | `text` | `Array<keyed object>` |
| `structuredObjectFieldModel` | `text` | `object` |
| `textFieldModel` | `text` | `string` |

`junctionFieldModel` powers `recordsField`-style relations and manages its own junction table instead of a column on the parent table. `matrixFieldModel` stores a JSON-serialized array of typed primitives in a regular `text` column.

Each model accepts options that fine-tune its behavior (for example, `textFieldModel({ allowEmptyString: true })`).

## Sync (migrations)

When you call `db.connect()` with `sync` enabled (the default), the ORM compares your collection definitions against the live schema and reconciles them:

1. Removes existing indexes and foreign keys.
2. Renames tables that share a `key` with one of your collections.
3. Resolves naming conflicts.
4. Optionally drops tables not in your collection map.
5. Creates missing tables.
6. Renames columns that share a `key`.
7. Resolves column name conflicts.
8. Optionally drops columns not in any field model.
9. Alters column types where it can be done without data loss.
10. Creates missing columns.
11. Rebuilds indexes and foreign keys.
12. Stores the new schema fingerprint in the `Options` table.

The whole thing runs inside a transaction on SQLite and PostgreSQL, and rolls back on errors. D1 cannot do this - back up before syncing.

### Disabling sync

```ts
new Database({ sync: false }) // Disabled entirely

new Database({
  sync: {
    dropNonCollectionTables: false, // Default: true
    dropNonFieldColumns: false,     // Default: true
  },
})
```

## Schema methods

The `Database` class also exposes manual schema methods. Most are unnecessary when `sync` is enabled.

```ts
await db.listTables()
await db.listColumns(tableName)
await db.listIndexes(tableName)
await db.listForeignKeys(tableName)
await db.tableExists(tableName)
await db.columnExists(tableName, columnName)
await db.indexExists(tableName, columnName)
await db.foreignKeyExists(tableName, columnName)
await db.createTable(tableName)
await db.createColumn(tableName, columnName, type, nullable)
await db.createIndex(tableName, columnNames, unique)
await db.createForeignKey(tableName, columnName, referencedTable, referencedColumn, action, transaction)
await db.renameTable(tableName, newTableName)
await db.renameColumn(tableName, columnName, newColumnName)
await db.changeColumnType(tableName, columnName, type, nullable)
await db.dropTable(tableName)
await db.dropColumn(tableName, columnName)
await db.dropIndex(indexName)
await db.dropForeignKey(tableName, constraintName, transaction)
```

## Raw queries

`db.exec(sql, params)` runs parameterized SQL with `$name` placeholders. Returns the rows for `SELECT` (and `RETURNING`) queries, or the number of affected rows otherwise.

```ts
await db.exec(`create table "Foo" (bar text)`)
await db.exec(`insert into "Foo" (bar) values ($bar)`, { bar: 'BAR' })
await db.exec(`select * from "Foo"`)
// [{ bar: 'BAR' }]
```

Parameters are escaped automatically.

### Transactions

Group queries into an atomic unit. Throws -> rollback. Supported on SQLite and PostgreSQL, **not** on D1.

```ts
await db.transaction(async (exec) => {
  await exec(`create table "Users" (name text, constraint "UX_Users__name" unique (name))`)
  await exec(`insert into "Users" (name) values ($name)`, { name: 'Admin' })
  await exec(`insert into "Users" (name) values ($name)`, { name: 'Admin' }) // duplicate -> rollback
})

await db.listTables() // ['Options']
```

## Query builder

`db.queryBuilder()` returns a fully-typed `QueryBuilder` instance that inherits your collection schema.

### Insert

```ts
const qb = db.queryBuilder()

const result = await qb.insertInto('Students')
  .values([
    { firstName: 'Harry', lastName: 'Potter', house: 1 },
    { firstName: 'Hermione', lastName: 'Granger', house: 1 },
  ])
  .returning(['firstName', 'lastName'])
  .run()

// result.success === true
// result.data: [{ firstName: 'Harry', lastName: 'Potter' }, ...]
```

If you omit `returning`, you get a count of inserted rows. Validation failures populate `result.inputErrors` with per-record field errors.

### Select

```ts
const students = await qb.selectFrom('Students')
  .select(['firstName', 'lastName'])
  .all()

const houses = await qb.selectFrom('Houses').selectAll().all()
```

`.first()` returns one row instead of all.

### Update

```ts
const updated = await qb.update('Students')
  .set({ house: 2, prefect: true })
  .where('firstName', '=', 'Harry')
  .returning(['firstName', 'house'])
  .run()
```

### Delete

```ts
const deleted = await qb.deleteFrom('Students')
  .where('firstName', '=', 'Draco')
  .returning(['firstName', 'lastName'])
  .run()
```

Every operation returns a `Result`-style object:

```ts
{
  success: boolean
  data: T | undefined
  runtimeError: string | undefined
  inputErrors: Record<string, string> | Record<string, string>[] | undefined
}
```

## Testing

The repo's test suite needs a PostgreSQL user that can create databases. Put credentials into `.env.test`:

```sh
VITE_PG_USER=pruvious
VITE_PG_PASSWORD=12345678
```

Run tests with `pnpm test`.
